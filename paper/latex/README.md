# Wraith paper — LaTeX source for arXiv submission

This folder contains a LaTeX version of the paper, generated from the
canonical Markdown source at `paper/wraith_paper_en.md`.

## Files

- `wraith_paper.tex` — main LaTeX file with preamble, title, author, abstract
  wrapper. Compile **this** file.
- `wraith_paper_body.tex` — body content generated by `pandoc` from the Markdown
  source. Included via `\input{wraith_paper_body.tex}` from the main file.
- `README.md` — this file.

## Regenerating the body from Markdown

The body is machine-generated. To regenerate after editing
`paper/wraith_paper_en.md`:

```bash
# From repo root, requires pandoc or pypandoc_binary (pip install pypandoc_binary)
python -c "
import pypandoc
pypandoc.convert_file(
    'paper/wraith_paper_en.md', 'latex', format='markdown',
    outputfile='paper/latex/wraith_paper_body.tex',
    extra_args=['--top-level-division=section'])
"
```

After regeneration, manually strip:
1. The initial `\section{Wraith: ...}` (title — already in `\maketitle`)
2. The author block
3. The first `\begin{center}\rule...\end{center}` horizontal rule
4. Convert the `\subsection{Abstract}` that follows into a proper
   `\begin{abstract} ... \end{abstract}` environment.

(The Python script in the repo's build helper does this automatically.)

## Compilation

Standard LaTeX pipeline (requires a LaTeX distribution like TeX Live or
MiKTeX):

```bash
cd paper/latex
pdflatex wraith_paper.tex
pdflatex wraith_paper.tex   # second pass for cross-references
```

Or with `latexmk`:

```bash
latexmk -pdf wraith_paper.tex
```

## Figure paths

Figures are referenced relative to `paper/charts/` via `\graphicspath`
in the preamble. No need to copy images — just compile from this
directory.

## Target categories for arXiv

- **Primary**: `cs.CL` (Computation and Language)
- **Cross-list**: `cs.LG` (Machine Learning), `cs.AI` (Artificial
  Intelligence)

## Submission checklist

Before uploading to arXiv:

- [ ] Compile locally without errors (`pdflatex wraith_paper.tex` twice)
- [ ] Check figures render correctly (look for `Missing file` warnings)
- [ ] Verify abstract wraps inside `\begin{abstract}...\end{abstract}`
- [ ] Confirm all URLs in text are wrapped in `\url{...}` or `\href{...}{...}`
- [ ] Ensure provenance table / every cited external number has a reference
- [ ] Endorsement email sent to a `cs.CL` endorser and reply received
- [ ] License selected on arXiv: `CC BY-NC-SA` (matches repo license)

## Known caveats of this conversion

- Pandoc converts Markdown tables to LaTeX `longtable`; rendering should be
  fine but may need minor adjustments for wide tables.
- Inline math `$...$` and display math `$$...$$` are preserved as-is.
- Some Unicode characters (×, →, etc.) may need `\usepackage{textcomp}`
  or manual replacement if they render as missing glyphs.
- Bibliography is inline (narrative citations); no separate `.bib` file.
  For final arXiv submission consider migrating to BibTeX.
